Program overview

The program starts on wednseday 31 october 2018 (see Kick-off! under) and will last 16 weeks. The technical skills teached in this program will follow a defined progression articulated around three corner stones: fundamental programming, web application development and software delivery. In the meantime the students will build up their soft-skills [...]. This page gives an overview of this periode of time.

Kick off!

The program starts with a particuliar session as known as kick-off. The kick-off will be an event in itself. The practical details of your venue and attendees will be available here

During this session we will set up your developper’s environment or check it out if you already finished with it (ref install your dev environment)

We will present all the tools we will use to work together on site and online

Fundamental programming

The aim of this module is to harmonize the group’s technical skills for programming. We use Javascript, first in the browser, and focus on the language itself.

At the end of this module the students shall be able to build a simple front-end application in a profesional software developer environment.

In details:

  • Javascript #1
  • Javascript #2
  • Javascript #3
  • Styling (HTML & CSS)
  • Source code management and collaboration around the code

Web application development

We start moving towards the server in this module. We will be using Javascript on the server for two reasons: #1 keep on learning of the language and #2 re-use the knowledge of the first module to be able to faster start server-side development

By completing this module, students should be able to write simple services on the server that will later be reused with their front-end code of the first module.

In details:

  • Architecture (HTTP, Rest, …)
  • Design of a webapp
  • Server-side development with Javascript
  • Persistence on the server (DBMS)
  • Hackathon
  • Team work and project management

Software delivery

This module will set all the previous work in a global perspective: a developer builds software for someone. It means that the students will have to work with concepts like requirements, testing, quality, build, release, and delivery. They will also enhance their technical skills on topics like systems, security and cloud computing

As a final assignment the students will present the result of their work: a state-of-the-art online web application

In details:

  • Build pipeline
  • Case
  • Security of a web application
  • Software delivery
  • Cloud computing
